# 메타 요소

이 문서는 게임 시스템 외부의 구조적 요소들을 정의한다.
게임 시작 설정, 수익화 모델, UI 설계 방향을 다룬다.

---

## 1. 게임 시작 설정

### 이야기꾼 선택

- 게임 시작 전에 이야기꾼을 선택한다 (`돌발_서사_요소.md` 참조)
- 이야기꾼은 돌발 사건의 빈도·강도·성향을 결정한다

### 사장 출신 배경 선택

- 게임 시작 전에 플레이어가 조작하는 **사장 캐릭터의 출신 배경**을 선택한다
- 출신 배경에 따라 초기 보유 자원, 인맥, 특기 등에 차이가 생긴다
- 출신 배경의 구체적인 종류와 효과는 추후 정의한다

### 게임 시작 시 결정 사항 요약

| 항목 | 설명 |
|------|------|
| 이야기꾼 | 돌발 서사 빈도·성향 선택 |
| 사장 출신 배경 | 초기 상태 및 특기 결정 |
| 난이도 | 이야기꾼 선택과 연동. 게임 중 변경 가능 |

---

## 2. 수익화 모델

### 핵심 원칙

- 수익화 요소는 **창작 루프를 절대 끊지 않는다**
- 유료 요소는 창작 루프 **외부**에만 위치한다
- 돈이 창작 결과에 영향을 주지 않는다 (선택의 책임이 플레이어에게 남는다)

### 수익화 방식

| 방식 | 내용 |
|------|------|
| 확장팩 유료 판매 | 새로운 아이돌 지망생 캐릭터를 추가하는 유료 앱 형태로 제공 |
| 외형·꾸미기 | 사장 캐릭터, 연습실, UI 테마 등의 꾸미기 요소 유료 제공 |
| 후원 | 후원(donation) 방식 수익화도 검토 |

### 금지 사항

- 창작 루프에 직접 개입하는 유료 요소 (예: 돈으로 스탯 향상, 사건 취소 등)는 허용하지 않는다
- 광고를 통한 수익화는 창작 몰입을 끊을 수 있어 지양한다

## 3. UI 설계 방향

### 핵심 원칙

- UI는 **창작 결정에 집중하게 만드는 도구**다
- 화려함보다 단순함과 기능 중심을 우선한다
- 케이팝 프로듀서 판타지의 분위기는 UI가 아닌 **콘텐츠와 서사**로 강화한다

### 정보 노출 방식

- 모든 정보를 한눈에 노출하지 않는다
- 근사치나 추측 가능한 **힌트 문구**를 통해 전략적 고민을 유도한다
  - 예: 스타크래프트에서 히드라리스크 덴 건물만 보고 히드라리스크를 대비하는 것과 같은 원리
- 탐색 자체도 하나의 플레이어 선택이다

### 모바일 최적화

- 플랫폼은 모바일(안드로이드)이다
- 복잡한 UI는 터치 조작의 답답함을 높이므로 배제한다
- 모든 결정과 조작은 터치 인터페이스에 최적화된 구조로 설계한다
- 조작 자체에서 불편함을 느끼지 않아야 창작 재미에 집중할 수 있다
