# 수치 체계 개요

이 문서는 게임 전반의 수치 설계 철학과 수치 간 연결 구조를 기술한다.

---

## 1. 수치 설계 철학

### 원칙 1: 수치는 결정의 근거이지 목적이 아니다

수치는 플레이어의 창작 결정(파트 배정, 협력자 선택, 연습량 조정)에 **의미 있는 피드백**을 주기 위해 존재한다.  
수치를 최대화하는 것 자체가 목표가 되지 않도록 설계한다.

### 원칙 2: 완전한 정보는 주지 않는다

- 스탯은 수치로 존재하지만 **처음부터 정확히 공개되지 않는다**
- 플레이어는 힌트·근사치를 통해 추론하며 결정을 내린다
- 상담 횟수가 쌓일수록 수치 정보가 정밀해진다

### 원칙 3: 수치는 단선이 아닌 관계다

- 한 수치가 다른 수치에 영향을 준다 (스탯 → 소화 밸류 → 파트 밸류 → 곡 밸류 → 앨범 밸류)
- 에너지·멘탈 같은 상태 자원은 스탯과 별개로 퍼포먼스 결과에 영향을 미친다

## 2. 수치 구조 전체 맵

```
[ 멤버 스탯 ]
  가창 스탯 (랩스킬, 음정, 성량, 리듬감, 감성, 음색)
  안무 스탯 (파워, 리듬감, 정확도, 유연성, 표현력, 스태미나)
  기타 스탯 (비주얼, 카리스마, 멘탈, 체력)
        │
        ▼
[ 멤버 소화 밸류 ]  ←  파트 블럭 요구 레벨(최소~최대) 대비 멤버 스탯 충족도
        │
        ▼
[ 파트 블럭 최종 밸류 ]  =  곡 밸류 + 안무 밸류 + 멤버 소화 밸류
        │
        ▼
[ 곡 최종 밸류 ]  =  모든 파트 블럭 최종 밸류의 합산
        │
        ├─→ MV 밸류 보정 (뮤직비디오 존재 시 추가)
        │
        ▼
[ 앨범 최종 밸류 ]  =  수록곡 최종 밸류의 합산
        │
        ▼
[ 공개 성과 ]  →  인지도 상승, 팬덤 규모 변화
```

## 3. 수치 범위 공통 기준

모든 수치는 게임 내 일관성을 위해 아래 단위 체계를 따른다.

| 수치 유형 | 범위 | 단위 개념 |
|-----------|------|-----------|
| 스탯 (가창·안무·기타) | 1 ~ 100 | 정수 |
| 파트 요구 레벨 | 1 ~ 100 | 정수 (최소~최대로 구간 표기) |
| 소화 밸류 | 0.0 ~ 1.5 | 소수 (1.0 = 완전 충족, 1.5 = 초과 퍼포먼스) |
| 파트 블럭 기본 밸류 (곡+안무) | 10 ~ 1000 | 정수 |
| 파트 블럭 최종 밸류 | 파트 기본 밸류 × 소화 밸류 | 소수 → 최종 반올림 |
| 곡 밸류 | 파트 블럭 최종 밸류 합산 | 정수 |
| 앨범 밸류 | 수록곡 밸류 합산 | 정수 |
| 에너지 | 0 ~ 최대치(체력 연동) | 정수 |
| 팬덤 규모 | 0 ~ 무제한 | 정수 (단위: 명) |
| 인지도 | 0 ~ 무제한 | 정수 (단위: pt) |
