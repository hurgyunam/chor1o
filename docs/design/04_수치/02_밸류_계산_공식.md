# 밸류 계산 공식

이 문서는 파트 블럭 밸류, 곡 밸류, 앨범 밸류의 산출 공식을 정의한다.  
소화 밸류 계산은 [멤버 스탯 수치](./01_멤버_스탯_수치.md)를 참조한다.

---

## 1. 파트 블럭 밸류

파트 블럭 밸류는 크게 세 구성 요소의 합산이다.

```
파트 블럭 최종 밸류 = (곡 밸류 + 안무 밸류) × 멤버 소화 밸류
```

### 1-1. 곡 밸류

작곡팀이 파트 블럭에 부여하는 기본 품질 수치이다.

| 결정 요인 | 설명 |
|-----------|------|
| 작곡팀 실력 | 실력이 높을수록 기본 곡 밸류 상단이 높아짐 |
| 파트 길이 (초) | 길수록 기본 밸류가 높아짐 (단, 선형 비례는 아님) |
| 돌발 결과물 여부 | 예상 장르 이탈 시 ±20% 보정 발생 (히트 또는 미스) |

**곡 밸류 범위 (파트 블럭 단위)**

| 작곡팀 실력 등급 | 파트당 곡 밸류 범위 |
|----------------|-------------------|
| 하 (실력 1~30) | 10 ~ 80 |
| 중 (실력 31~60) | 50 ~ 200 |
| 상 (실력 61~85) | 150 ~ 400 |
| 최상 (실력 86~100) | 350 ~ 600 |

- 파트 길이 보정: `곡 밸류 × (파트 길이(초) / 15초)^0.6`  
  (15초를 기준으로 짧으면 감소, 길면 증가하되 체감 증가율은 점점 낮아진다)

### 1-2. 안무 밸류

안무팀이 파트 블럭에 부여하는 안무 품질 수치이다.  
안무 의뢰 없이 진행한 경우 안무 밸류는 **0**이다.

| 결정 요인 | 설명 |
|-----------|------|
| 안무팀 실력 | 실력이 높을수록 기본 안무 밸류 상단이 높아짐 |
| 파트 길이 (초) | 곡 밸류와 동일한 보정 적용 |
| 가창없음 파트 | 안무 밸류만 존재하는 순수 퍼포먼스 파트 가능 |

**안무 밸류 범위 (파트 블럭 단위)**

| 안무팀 실력 등급 | 파트당 안무 밸류 범위 |
|----------------|---------------------|
| 하 (실력 1~30) | 5 ~ 60 |
| 중 (실력 31~60) | 40 ~ 160 |
| 상 (실력 61~85) | 120 ~ 320 |
| 최상 (실력 86~100) | 280 ~ 480 |

### 1-3. 파트 블럭 최종 밸류 계산 예시

```
예시:
  파트 길이: 20초
  작곡팀 실력: 55 → 곡 밸류 기본 120
  파트 길이 보정: 120 × (20/15)^0.6 ≈ 120 × 1.27 ≈ 152
  안무팀 실력: 48 → 안무 밸류 기본 90
  안무 밸류 파트 보정: 90 × 1.27 ≈ 114

  곡+안무 합산: 152 + 114 = 266
  멤버 소화 밸류: 0.85
  파트 블럭 최종 밸류: 266 × 0.85 ≈ 226
```

## 2. 곡 밸류

### 기본 계산

```
곡 밸류 = Σ 모든 파트 블럭 최종 밸류
```

### MV 보정

뮤직비디오가 존재하는 경우 곡 밸류에 추가 보정이 적용된다.

```
MV 보정 후 곡 밸류 = 곡 밸류 × (1 + MV 보정 계수)
```

| MV 촬영팀 실력 등급 | MV 보정 계수 |
|--------------------|------------|
| 없음 | 0 |
| 하 (실력 1~30) | 0.03 ~ 0.07 |
| 중 (실력 31~60) | 0.08 ~ 0.15 |
| 상 (실력 61~85) | 0.16 ~ 0.25 |
| 최상 (실력 86~100) | 0.26 ~ 0.35 |

- MV는 출시 전 공개 또는 후공개 시점에 무관하게 동일한 보정 계수를 적용한다
- MV 보정은 곡이 **공개된 이후** 성과 계산에 반영된다

### 곡 인지도 환산

```
곡 인지도 획득 = 곡 밸류 × 팬덤 몰입력 계수 × 출시 방식 계수
```

| 출시 방식 | 출시 방식 계수 |
|-----------|--------------|
| 음원 등록 | 0.8 |
| 피지컬 앨범 (포함 곡) | 1.2 |

## 3. 앨범 밸류

### 기본 계산

```
앨범 밸류 = Σ 수록곡 밸류
```

### 타이틀곡 가중치

앨범에는 하나 이상의 타이틀곡이 있으며, 타이틀곡은 앨범 밸류에 추가 가중치를 부여한다.

```
앨범 최종 밸류 = (수록곡 밸류 합산) + (타이틀곡 밸류 합산 × 0.3)
```

- 타이틀곡이 여러 개일 경우 모든 타이틀곡에 0.3 가중치 적용
- 타이틀곡 선정은 플레이어의 전략적 판단에 따른다 (가장 높은 밸류의 곡이 반드시 타이틀일 필요 없음)

### 앨범 인지도 환산

```
앨범 인지도 획득 = 앨범 최종 밸류 × 팬덤 몰입력 계수 × 출시 방식 계수
```

| 출시 방식 | 팬덤 영향 | 인지도 영향 |
|-----------|----------|------------|
| 음원 등록 | 낮음 | 보통 |
| 피지컬 앨범 | 높음 (팬덤 몰입력 상승) | 높음 |

## 4. 잭팟 보정 (랜덤 폭발)

> 플레이어의 도파민을 자극하기 위해, 낮은 확률로 밸류 수치가 비정상적으로 폭발하는 **잭팟 이벤트**를 계획하고 있다.  
> 안정적인 성장 곡선을 따라가다가 예상 밖의 대성공이 터지는 경험이 핵심 동기다.

### 4-1. 잭팟 발생 구조

잭팟은 **파트 블럭 밸류 확정 시점**과 **앨범 인지도 환산 시점** 두 단계에서 각각 독립적으로 발생할 수 있다.

```
잭팟 발생 여부: 매 계산마다 독립 확률 판정
잭팟 발생 시: 해당 단계의 밸류 또는 인지도 획득량에 잭팟 배율 적용
```

| 잭팟 등급 | 발생 확률 | 배율 |
|-----------|-----------|------|
| 소형 잭팟 | 약 10% | × 1.5 ~ 2.0 |
| 중형 잭팟 | 약 3% | × 2.0 ~ 4.0 |
| 대형 잭팟 | 약 0.5% | × 4.0 ~ 10.0 |

- 잭팟은 연속 발생이 가능하다 (파트 블럭마다 독립 판정)
- 파트 블럭 잭팟이 여러 개 겹치면 곡 밸류 전체가 폭발적으로 상승하는 시너지가 생긴다

### 4-2. 잭팟 확률 변동 요인

플레이어의 선택에 따라 잭팟 확률을 소폭 높이거나 낮출 수 있다.  
단, 확률을 높이는 선택은 대개 **기대 평균값을 낮추는 리스크**를 동반한다.

| 요인 | 잭팟 확률 변화 |
|------|--------------|
| 돌발 서사 적극 수용 | +1 ~ 3% |
| 실험적 장르 선택 | +2% (단, 미스 시 밸류 -30%) |
| 신인 협력자 기용 | +1% (단, 기본 밸류 범위 하단) |
| 안정 노선 유지 | 변화 없음 (기대 평균값 보장) |

### 4-3. 잭팟 연출 의도

- **예측 불가능성**: 플레이어가 어떤 선택을 하더라도 잭팟 발생 여부를 사전에 알 수 없다
- **서사적 흥분**: 대형 잭팟 발생 시 인게임 이벤트 연출을 통해 "바이럴 히트" 순간을 극적으로 표현할 예정
- **중독성 설계**: 슬롯머신과 유사한 가변 비율 강화 스케줄을 참조하되, 플레이어가 통제감을 잃지 않도록 전략적 선택과 연계한다

## 5. 팬덤·인지도 수치

### 팬덤 규모

- 팬덤 규모는 **정수 (명)** 로 표현된다
- 앨범 출시, 무대 활동, MV 조회수 누적에 따라 증가한다
- 돌발 사건(스캔들, 멤버 이탈 위협 등)으로 일시적 감소 가능

### 팬덤 몰입력

- 팬덤 규모와 별개인 **질적 지표**이다 (범위: 0.5 ~ 2.0)
- 피지컬 앨범 구매·팬미팅 참여 등 높은 참여 행동에 의해 상승
- 몰입력이 높을수록 동일한 밸류에서 인지도 획득량이 더 크다

### 인지도

- 누적 정수 (pt 단위)
- 인지도 구간에 따라 접근 가능한 협력자 풀의 수준이 달라진다

| 인지도 구간 | 접근 가능 협력자 등급 |
|------------|---------------------|
| 0 ~ 999 | 실력 하 협력자만 접근 가능 |
| 1,000 ~ 4,999 | 실력 중 협력자 접근 가능 |
| 5,000 ~ 19,999 | 실력 상 협력자 접근 가능 |
| 20,000 이상 | 실력 최상 협력자 접근 가능 |
