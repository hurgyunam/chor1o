# 멤버 스탯 수치

이 문서는 멤버 스탯의 초기값 범위, 성장 수치, 소화 밸류 계산 방식을 정의한다.  
스탯 개념 정의는 [멤버 요소](../02_개념/02_멤버_요소.md)를 참조한다.

---

## 1. 스탯 초기값

### 연습생 초기 스탯 범위

| 가창 스탯 범위 | 안무 스탯 범위 | 기타 스탯 범위 |
|---------------|---------------|---------------|
| 5 ~ 35 | 5 ~ 35 | 10 ~ 50 |

- 모든 멤버는 연습생으로 시작하며, 데뷔 후 정식 멤버 스탯은 연습생 시절 성장 결과가 그대로 이어진다
- 각 스탯은 독립적으로 난수 생성된다 (모든 스탯이 동일하지 않다)
- 한 멤버가 전 스탯에서 골고루 높은 것은 불가능하도록 설계한다 → **전문성 유도**
- 멤버 생성 시 1~2개의 스탯은 해당 멤버의 **특기 스탯**으로 범위 상단에 치우쳐 생성된다

### 잠재력 수치

- 연습생 멤버는 보이지 않는 **잠재력** 수치를 보유한다 (범위: 0.5 ~ 2.0)
- 잠재력이 높을수록 같은 연습량 대비 스탯 성장 속도가 빠르다
- 잠재력은 초기 상담 누적을 통해 힌트 수준으로만 파악 가능하다

## 2. 스탯 성장

### 성장 원천

| 성장 원천 | 성장 대상 스탯 | 1회 성장 수치 |
|-----------|---------------|---------------|
| 노래연습 | 가창 스탯 전체 (활용 스탯에 비례) | 0 ~ 3 |
| 안무연습 | 안무 스탯 전체 (활용 스탯에 비례) | 0 ~ 3 |
| 녹음 경험 | 가창 스탯 중 활용된 스탯 | 0 ~ 1 |
| 무대 퍼포먼스 | 가창·안무·카리스마 | 0 ~ 2 |
| 상담 | 멘탈 | 0 ~ 5 |
| 휴식 | 멘탈·체력 (소폭 회복) | 0 ~ 2 |
| 돌발 사건 (부정) | 멘탈 (감소) | -1 ~ -10 |
| 돌발 사건 (긍정) | 카리스마·비주얼 | 0 ~ 3 |

- 성장 속도 밸런스의 목표: **"지루하지 않되, 컨트롤 불가 느낌도 없는"** 구간 유지
- 판단 기준점 (밸런스 테스트 시 참조): 싱글 1곡 사이클 기준 주력 스탯 평균 +3 ~ +8 수준, 시즌 전반부(1~3회차) 내 주력 스탯이 60~70 도달이 이상적인 페이스로 간주

### 성장 계산식

```
실제 성장량 = 기본 성장량 × 잠재력 × 연습 강도 보정
```

- **기본 성장량**: 성장 원천별 위 표의 범위 내 난수
- **잠재력**: 해당 멤버의 잠재력 수치 (0.5 ~ 2.0)
- **연습 강도 보정**: 에너지 소모를 감수하고 고강도 연습 선택 시 최대 1.5 적용

### 스탯 상한

- 모든 스탯의 최대값은 **100**
- 100에 가까울수록 성장 속도가 점진적으로 감소 (수렴 구조)
  - 스탯 1 ~ 50: 성장 보정 없음
  - 스탯 51 ~ 75: 성장량 × 0.7
  - 스탯 76 ~ 90: 성장량 × 0.4
  - 스탯 91 ~ 100: 성장량 × 0.15

## 3. 소화 밸류 계산

소화 밸류는 **멤버의 스탯이 파트 블럭의 요구 레벨을 얼마나 충족하는가**를 나타내는 수치다.  
파트 블럭에는 최소 요구 레벨과 최대 요구 레벨(이상적 레벨)이 있다.

- 소화 밸류 배정의 설계 의도: 플레이어가 **이상적인 선택을 포기하고 차선을 선택하는 상황**을 만드는 것
- 판단 기준점 (밸런스 테스트 시 참조): 파트 배정 시 "이상적 소화(1.0 이상)" 멤버가 1~2명, "보통 소화(0.5~0.79)" 멤버가 과반인 상태가 플레이어에게 적절한 긴장감을 유발하는 이상적인 구간으로 간주
- 최소·최대 요구 레벨의 간격이 너무 좁으면 "다 별로" / 너무 넓으면 "아무나 써도 됨" 상황이 발생하므로, 요구 레벨 간격 설계는 이 의도를 기준으로 검증한다

### 파트 요구 레벨 구조

| 속성 | 설명 |
|------|------|
| 최소 요구 레벨 | 이 레벨 미만이면 파트를 소화하지 못함 (소화 밸류 0.0 ~ 0.5) |
| 최대 요구 레벨 | 이 레벨 이상이면 최고 소화 상태 (소화 밸류 1.0 ~ 1.5) |

### 소화 밸류 산출 방식

파트 블럭은 여러 스탯을 요구하며, 각 스탯의 충족도를 **장르에 따른 가중 평균**으로 합산한다.

#### 단계 1: 개별 스탯 충족도 산출

```
스탯 충족도 = (멤버 스탯 - 최소 요구 레벨) / (최대 요구 레벨 - 최소 요구 레벨)
```

| 충족도 범위 | 해석 |
|-------------|------|
| 충족도 < 0 | 최소 요구 미달 |
| 0 ≤ 충족도 < 1 | 부분 충족 |
| 충족도 = 1 | 이상적 충족 |
| 충족도 > 1 | 초과 역량 (일부 장르에서 보정 적용) |

#### 단계 2: 장르별 가중 평균

각 파트 블럭은 장르에 따라 활용 스탯이 다르며, 아래처럼 가중치를 갖는다.

**가창 장르별 스탯 가중치 예시**

| 장르 | 랩스킬 | 음정 | 성량 | 리듬감 | 감성 | 음색 |
|------|--------|------|------|--------|------|------|
| 랩 | 0.4 | 0.0 | 0.1 | 0.3 | 0.0 | 0.2 |
| R&B | 0.0 | 0.35 | 0.1 | 0.1 | 0.35 | 0.1 |
| 정통 발라드 | 0.0 | 0.35 | 0.3 | 0.05 | 0.25 | 0.05 |
| 포크 | 0.0 | 0.3 | 0.1 | 0.1 | 0.1 | 0.4 |
| 댄스팝 | 0.0 | 0.3 | 0.1 | 0.4 | 0.1 | 0.1 |
| 시티팝 | 0.0 | 0.2 | 0.05 | 0.1 | 0.3 | 0.35 |

**안무 장르별 스탯 가중치 예시**

| 장르 | 파워 | 리듬감 | 정확도 | 유연성 | 표현력 | 스태미나 |
|------|------|--------|--------|--------|--------|--------|
| 힙합 | 0.35 | 0.35 | 0.1 | 0.05 | 0.1 | 0.05 |
| 걸크러시 | 0.35 | 0.1 | 0.3 | 0.05 | 0.1 | 0.1 |
| 청순 | 0.05 | 0.15 | 0.2 | 0.3 | 0.25 | 0.05 |
| 퍼포먼스 | 0.1 | 0.1 | 0.1 | 0.1 | 0.35 | 0.25 |
| 팝 | 0.1 | 0.3 | 0.35 | 0.1 | 0.1 | 0.05 |

#### 단계 3: 소화 밸류 확정

```
원시 소화 밸류 = Σ (스탯 충족도 × 해당 스탯 가중치)
```

| 원시 소화 밸류 | 최종 소화 밸류 | 퍼포먼스 상태 |
|----------------|----------------|--------------|
| < 0 | 0.0 | 파트 소화 실패 |
| 0 ~ 0.49 | 0.0 ~ 0.49 | 미숙한 소화 |
| 0.5 ~ 0.79 | 0.5 ~ 0.79 | 보통 소화 |
| 0.8 ~ 0.99 | 0.8 ~ 0.99 | 안정적 소화 |
| 1.0 ~ 1.19 | 1.0 ~ 1.19 | 이상적 소화 |
| ≥ 1.2 | 1.2 ~ 1.5 (상한) | 초과 역량 발휘 (장르에 따라 보정) |

- 초과 역량(1.0 이상)은 카리스마·비주얼 기타 스탯의 영향으로 최대 **1.5**까지 확장된다
- 기타 스탯 보정: `최종 소화 밸류 += (카리스마 / 100) × 0.1 + (비주얼 / 100) × 0.05`

## 4. 스탯 공개 진행도

| 상담 횟수 (멤버 1인 기준) | 공개 수준 |
|--------------------------|-----------|
| 0회 | 스탯 존재만 암시 (힌트 문구 표시 불가) |
| 1 ~ 2회 | "높음 / 보통 / 낮음" 3단계 수준 표시 |
| 3 ~ 5회 | ±10 오차의 근사 수치 표시 |
| 6 ~ 9회 | ±5 오차의 근사 수치 표시 |
| 10회 이상 | 정확한 수치 공개 |
