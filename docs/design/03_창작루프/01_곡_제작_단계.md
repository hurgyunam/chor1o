# 곡 제작 단계

이 문서는 창작 루프의 PHASE 1~4에 해당하는 **곡 제작 과정**을 단계별로 정의한다.
작곡 의뢰부터 녹음 완료까지, 각 단계에서 플레이어가 내리는 결정과 대기 구조를 기술한다.

곡 상태 및 파트 블럭 속성 정의는 [`01_창작_요소.md`](../02_개념/01_창작_요소.md)를 참조한다.

---

## PHASE 1: 작곡 의뢰

### 진입 조건
- 미완성 상태의 곡이 존재해야 한다
- 충분한 자금이 있어야 한다

### 플레이어 행동 흐름

```
작곡팀 목록 열람
      │
      ▼
작곡팀 선택 (속성 확인: 인지도, 평판, 장르 성향, 실력, 비용, 돌발 확률)
      │
      ▼
작곡팀이 제안 제시
  - 페이 (확정)
  - 파트 블럭 프리뷰 (가창 장르, 곡 밸류 범위 힌트, 파트 블럭 수)
      │
      ├─ [수락] → 페이 지불 → 대기 → 곡 수령 (곡 상태: 작곡완성)
      └─ [거절] → 다른 작곡팀 재탐색
```

### 결과물
- **파트 블럭** 생성 (가창 장르, 요구 가창 레벨, 곡 밸류, 시간 구간 확정)
- 안무 관련 속성은 아직 미확정 상태

### 플레이어 판단 포인트
- 제안된 파트 블럭 프리뷰가 우리 그룹 멤버 스탯에 맞는가
- 작곡팀의 돌발 확률이 감당 가능한 수준인가
- 비용 대비 기대 곡 밸류가 합리적인가

---

## PHASE 2: 안무 의뢰 (선택)

### 진입 조건
- 곡 상태가 **작곡완성** 이상이어야 한다
- 안무 의뢰는 필수가 아니다. 생략 시 안무 없이 파트 분배 단계로 진행한다

### 플레이어 행동 흐름

```
안무 의뢰 여부 결정
      │
      ├─ [생략] → PHASE 3으로 이동 (안무 밸류 없이 진행)
      │
      └─ [의뢰]
            │
            ▼
      안무팀 목록 열람
            │
            ▼
      안무팀 선택 (속성 확인: 인지도, 평판, 스타일 성향, 실력, 비용, 돌발 확률)
            │
            ▼
      안무팀이 제안 제시
        - 페이 (확정)
        - 파트 블럭 프리뷰 (안무 장르, 안무 밸류 범위 힌트)
            │
            ├─ [수락] → 페이 지불 → 대기 → 안무 수령 (곡 상태: 코레오완성)
            └─ [거절] → 다른 안무팀 재탐색 또는 생략 결정
```

### 결과물
- 기존 파트 블럭에 **안무 속성** 추가 (안무 장르, 요구 안무 레벨, 안무 밸류 확정)

### 플레이어 판단 포인트
- 안무 의뢰 비용이 기대 밸류 상승분을 정당화하는가
- 제안된 안무 스타일이 그룹 콘셉트 및 멤버 구성과 맞는가
- 안무팀의 돌발 결과물이 나왔을 때 대응 계획이 있는가

---

## PHASE 3: 파트 분배

### 진입 조건
- 곡 상태가 **작곡완성** 또는 **코레오완성** 상태이어야 한다

### 플레이어 행동 흐름

```
파트 블럭 목록 열람 (가창 장르, 요구 레벨, 시간 구간 등 확인)
      │
      ▼
각 파트 블럭에 담당 멤버 배정
  - 멤버 스탯 vs 파트 요구 레벨 비교
  - 멤버 에너지 소모량 누적 확인
  - 멤버 소화 밸류 예상치 확인 (불완전 정보: 스탯이 완전히 공개되지 않을 수 있음)
      │
      ▼
안무가 있는 경우: 포메이션 구성
  - 센터, 리드댄서, 메인댄서 배치 결정
      │
      ▼
파트 분배 확정 (곡 상태: 파트분배완료)
  - 멤버별 에너지 소모량 확정
  - 멤버 소화 밸류 및 최종 합산 밸류 생성 (지속 변동)
```

### 결과물
- 각 파트 블럭의 **담당 멤버** 확정
- 곡 전체의 **멤버별 에너지 소모량** 확정
- **멤버 소화 밸류** 생성 (연습에 따라 이후 변동)

### 플레이어 판단 포인트
- 특정 멤버에게 에너지 소모가 집중되지 않는가
- 스탯이 불완전하게 공개된 멤버에게 중요 파트를 맡겨도 되는가
- 상담 중인 멤버가 있다면 해당 멤버를 제외한 재구성이 필요한가

### 불완전 정보의 역할
파트 분배 시 멤버의 스탯이 완전히 공개되지 않을 수 있다.
상담 횟수가 적은 멤버는 스탯이 힌트 문구 수준으로만 보이므로,
플레이어는 불완전한 정보 속에서 판단을 내려야 한다.

## PHASE 4: 녹음 및 연습

### 진입 조건
- 곡 상태가 **파트분배완료** 이상이어야 한다

### 녹음 전 연습

녹음 전에 멤버들이 연습하면 파트 소화 밸류가 상승한다.
연습은 에너지를 소모하므로, 과도한 연습은 녹음 시 퍼포먼스를 오히려 떨어뜨릴 수 있다.

| 연습 유형 | 효과 | 에너지 소모 |
|-----------|------|-------------|
| 노래연습 | 가창 파트 소화 밸류 향상 | 있음 |
| 안무연습 | 안무 파트 소화 밸류 향상 | 있음 |
| 휴식 | 에너지 회복 | 없음 (회복) |

### 녹음

```
녹음 실행
      │
      ▼
각 멤버의 현재 에너지 + 스탯 + 파트 소화 밸류를 기반으로 결과 확정
      │
      ▼
곡 상태: 파트분배완료 → 녹음완료
```

녹음 후 안무가 있는 경우, 안무숙지 과정이 추가로 진행된다.

### 안무숙지 (안무가 있는 경우)

```
안무연습 진행
      │
      ▼
안무 소화 수준이 기준 이상 달성 시 → 곡 상태: 안무숙지완료
```

안무숙지가 완료되지 않은 상태로도 공개는 가능하나, 퍼포먼스 품질 저하가 발생한다.

### 플레이어 판단 포인트
- 녹음 전 연습에 얼마나 에너지를 투자할 것인가
- 에너지가 부족한 멤버를 먼저 휴식시킬 것인가
- 안무숙지 완료 전 출시를 강행할 것인가 (빠른 출시 vs 품질)
